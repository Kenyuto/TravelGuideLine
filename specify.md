# 📌 **旅遊行程檢視網站 — 需求面總整理（含補充）**

## ✅ **核心功能需求**

### 1. **行程資料來源**

* 網站需能讀取指定 Google Sheet 的資料並呈現。
* 支援資料快取/更新機制，確保行程更新後使用者能看到最新內容。
  -（補充）需能處理 Google Sheet 欄位變動的容錯機制（欄位新增或調整時不要壞掉）。

---

### 2. **多裝置適應（RWD）**

* 支援手機、平板、桌機等不同螢幕尺寸。
  -（補充）橫向與縱向模式都需可正常顯示。
  -（補充）字體大小需可在小螢幕上清晰閱讀。

---

### 3. **介面與呈現（UX/UI）**

* 使用 Emoji 增加可讀性及趣味性，例如：

  * 📍景點
  * 🍽️ 餐廳
  * 🚌 交通
  * 🏨 住宿
  * 📅 日期 / 行程分類
* 支援清楚的行程切換方式（如分日頁籤、多日滑動切換）。
  -（補充）提供簡易的搜尋或過濾行程方式。
  -（補充）行程資訊需有一致的排版，例如卡片式呈現。

---

### 4. **免費部署**

* 網站架構與技術需可部署在免費平台上（例如 GitHub Pages、Cloudflare Pages、Vercel 的免費方案）。
  -（補充）需可搭配 Google Sheet API 或公開 CSV/JSON 取用的方式，不需額外付費後端。
  -（補充）網站需無後端或僅使用前端運算，以減少成本。

---

## 🔧 **補充：為確保網站能穩定運作而應具備的需求**

### 5. **效能與使用體驗**

* 載入畫面（Loading Animation）避免空白頁。
* Google Sheet 取用失敗時需顯示錯誤訊息。

---

### 6. **離線或弱網路體驗（可選但實用）**

-（補充）可加入基礎 PWA 支援（至少允許快取 CSS/JS），讓行程在弱網路也能瀏覽。

---

### 7. **SEO 與分享**

* 基礎的 meta 資料與社群預覽（讓使用者分享時好看）。
* 網站可透過網址直接開啟特定日期/行程頁（Deep Link）。

---

### 8. **安全與權限**

* Google Sheet 可設定為「只讀」公開或使用 API Key 方式安全取用。
  -（補充）避免將敏感金鑰直接暴露在前端。

---

## 🎨 **視覺與互動（強烈建議的需求）**

* 使用卡片式區塊呈現行程內容。
* 使用表格或時間軸方式呈現每日行程。
* 行程分段標註：

  * 時段：⏰
  * 交通：🚗/🚆/🚌
  * 重要提醒：⚠️
  * 價格資訊：💰
    -（補充）主題色、字體風格與排版需一致。

---

# ⭐ **總結：網站最終會需要具備的內容**

以下為精簡的必備項目清單：

### 📄 頁面內容

* 首頁（展示旅遊天數、快速入口）
* 每日行程頁（含景點/時間/圖片/emoji）
* 行程詳細頁（可選）
* 說明頁（說明如何使用）

### 📦 功能項目

* Google Sheet 資料匯入
* 日期與行程選擇 UI
* RWD 多裝置適配
* Emoji 視覺化呈現
* 無後端＆可免費部署
* Loading 狀態
* 錯誤提示
* SEO/分享設定（可選）
* PWA 快取（可選）
